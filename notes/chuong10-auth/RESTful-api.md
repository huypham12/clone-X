/*
  API (Application Programming Interface): Giúp các phần mềm giao tiếp với nhau thông qua các giao thức (như HTTP, WebSocket...).

  REST (Representational State Transfer): Một kiểu thiết kế API phổ biến – định nghĩa các nguyên tắc/ràng buộc để tổ chức giao tiếp client - server một cách đơn giản, rõ ràng, dễ mở rộng.
  (Ngoài REST còn có GraphQL, gRPC,... cũng dùng để thiết kế API)

  Một API REST tốt thường có những đặc điểm sau:

  - Đặt tên tài nguyên (resource) rõ ràng, hợp lý, dễ hiểu với người dùng API (ví dụ: /users, /orders/123).
  - Sử dụng đúng HTTP methods:
      GET    – Lấy dữ liệu
      POST   – Tạo mới
      PUT    – Cập nhật toàn bộ
      PATCH  – Cập nhật một phần
      DELETE – Xóa dữ liệu
  - Trả về đúng HTTP status code để thể hiện kết quả:
      200 OK, 201 Created, 400 Bad Request, 404 Not Found, 500 Internal Server Error...
  - Giao tiếp bằng định dạng JSON (thông dụng, dễ xử lý).
  - Tách biệt giữa client - server (bạn làm việc bạn tôi làm việc tôi)
*/
